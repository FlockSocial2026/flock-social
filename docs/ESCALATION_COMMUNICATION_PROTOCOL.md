# Escalation Communication Protocol

## Severity Definitions
- Sev 1: Trust-critical / launch-blocking
- Sev 2: Major conversion or pilot execution risk
- Sev 3: Non-critical process/content issue

## Communication SLAs
- Sev 1: Immediate alert + decision within 1 hour
- Sev 2: Alert same day + mitigation within 24 hours
- Sev 3: Batch into daily review

## Message Format
1. Issue summary
2. Impact scope
3. Proposed mitigation
4. Owner + deadline
5. Confirmation of action start
