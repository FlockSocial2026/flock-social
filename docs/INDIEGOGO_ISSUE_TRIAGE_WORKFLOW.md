# Indiegogo Issue Triage Workflow

## Intake Channels
- Campaign comments/messages
- Outreach replies
- Metrics anomalies
- Internal QA observations

## Severity Levels
- P1: Launch-blocking or trust-critical issue
- P2: Conversion-impacting issue
- P3: Minor copy/process issue

## Workflow
1. Log issue with timestamp and source
2. Assign severity + owner
3. Define mitigation action
4. Execute fix and confirm effect
5. Document in end-of-day log

## SLA Targets
- P1: immediate action (<1h)
- P2: same-day resolution
- P3: batch in next daily cycle
