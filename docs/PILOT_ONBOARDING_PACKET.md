# Pilot Onboarding Packet (Church Cohort v1)

## Packet Contents
1. Product one-pager (mission + value)
2. Setup checklist (church bootstrap + admin access)
3. Weekly operating cadence template
4. KPI glossary + reporting cadence
5. Support/escalation contact path

## Church Kickoff Checklist
- [ ] Create church via `/flock` bootstrap
- [ ] Confirm church admin role
- [ ] Add pastor/staff roles
- [ ] Publish first weekly announcement
- [ ] Create first 2-3 events
- [ ] Invite initial member cohort
- [ ] Confirm metrics visibility in `/flock/admin`

## Week 1 Success Criteria
- At least 1 weekly push sent
- At least 1 event created with RSVPs
- At least 10 members onboarded in pilot church
- Metrics endpoint reporting non-zero church activity

## Reporting Rhythm
- Weekly summary exported via `/api/metrics/pilot-summary?churchId=<id>&format=csv`
- Weekly qualitative feedback note appended to pilot log
