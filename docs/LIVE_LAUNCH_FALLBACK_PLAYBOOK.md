# Live Launch Fallback Playbook

## Scenario A — Critical link failure at/after launch
1. Pause outbound sends
2. Patch link in canonical assets
3. Revalidate links on mobile/desktop
4. Resume sends with correction note

## Scenario B — Media render/upload issue
1. Swap to fallback approved media set
2. Continue launch flow without blocking timeline
3. Log incident and backfill final media later same day

## Scenario C — Severe conversion underperformance (first 6h)
1. Shift to highest-performing channel immediately
2. Deploy alternative CTA variant
3. Publish clarification/update post to address objections

## Scenario D — Response backlog overload
1. Enter response sprint mode
2. Prioritize high-intent inquiries
3. Use objection-response bank macros
